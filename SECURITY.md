# Security Policy

## Supported versions

Security fixes are applied to the latest release of each public Cross Clinical OSS repository.

## Reporting a vulnerability

Please **do not** open a public issue for security vulnerabilities.

Email: security@crossclinical.com (or open a private GitHub Security Advisory on the affected repo).

## PHI and sensitive data

Never commit, upload, or paste protected health information (PHI), credentials, or real patient records into issues, PRs, Spaces, or fixtures. Educational and synthetic data only.
